Shortest Path (Volunteer -> Accepted Request) Implementation
===========================================================

Backend required?
-----------------
For the current requirement (draw shortest path line between 2 points), NO backend change is required.
It is implemented fully on frontend by requesting route geometry from OSRM public routing API.

When backend WOULD be needed (optional future):
- If you want your own private routing service/proxy
- If you want API-key protected routing provider (Google/Mapbox/etc.) managed server-side
- If you want route caching, analytics, quota control, or custom route rules

Flow / Trigger points
---------------------
Trigger A: Volunteer accepts request
1. Home controller accepts request and navigates to map:
   Get.toNamed(RouteNames.map, arguments: {'request': activeRequest.toJson()})
2. MapScreen init:
   mapCntrl.setActiveRequestFromArguments(Get.arguments)
3. In MapCntrl:
   setActiveRequest(...) stores active request + calls _scheduleRouteRefresh(force: true)

Trigger B: First live location received
1. _requestLocationPermission() gets current GPS location
2. currentLatLng is set
3. _scheduleRouteRefresh(force: true) is called
4. If both points exist (current + request), shortest route fetch starts

Trigger C: Volunteer keeps moving
1. _startPositionStream() updates currentLatLng continuously
2. each update calls _scheduleRouteRefresh()
3. refresh is throttled:
   - skip if last fetch is very recent (<12s) AND movement <=35m
   - avoids spamming route API

Trigger D: Map rendering
1. MapScreen passes mapCntrl.activeRoutePoints into buildMap(...)
2. build_map.dart already has PolylineLayer using routePoints
3. route displayed on map


Routing request details
-----------------------
Endpoint called from frontend:
https://router.project-osrm.org/route/v1/driving/{fromLng},{fromLat};{toLng},{toLat}?overview=full&geometries=geojson

Behavior:
- On success: decode GeoJSON coordinates into LatLng list and draw full road path
- On failure: fallback to simple 2-point line (current -> target)

Summary
-------
You can run this feature as-is with frontend only.
No backend changes are mandatory for "draw shortest path between volunteer and request points".
